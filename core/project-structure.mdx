## Directory Structure

The Doofus project follows the Next.js 16 App Router convention:

```text
doofus-v1/
├── src/
│   └── app/          # App Router directory
│       ├── globals.css   # Global styles
│       ├── layout.tsx    # Root layout
│       └── page.tsx      # Home page
├── public/           # Static files
├── next.config.ts    # Next.js configuration
├── package.json      # Project dependencies
├── tsconfig.json     # TypeScript configuration
└── biome.json       # Biome configuration
```

## Key Files

### src/app/layout.tsx
The root layout component that wraps all pages. This is where global providers and layouts are defined.

### src/app/page.tsx
The main page component. This is the first page users see when visiting the application.

### src/app/globals.css
Global styles and Tailwind directives.

### next.config.ts
Configuration file for Next.js, written in TypeScript.

### biome.json
Configuration for the Biome toolchain, handling both linting and formatting.
